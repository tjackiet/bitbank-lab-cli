# ADR-001: CLI と MCP サーバーを別リポジトリで管理する

## ステータス

Accepted

## コンテキスト

bitbank-genesis-mcp-server（現: bitbank-lab-mcp）（MCP サーバー、22 ツール）が既に存在する。CLI × Skills スターターキットを新たに作るにあたり、同一リポか別リポかの判断が必要になった。

## 決定

別リポジトリ（bitbank-lab-cli）で管理する。

## 理由

- **ターゲットユーザーが違う。** MCP は汎用 LLM ユーザー向け、CLI は中上級システムトレーダー向け
- **README・セットアップ手順・Issue の文脈がまったく異なる。** 混在するとどちらのユーザーにも不親切
- **CLI 側の変更が MCP の CI を壊すリスクがない。** デプロイサイクルを独立に保てる
- **テスターに「このリポを clone して `npx bitbank ticker btc_jpy` を試して」と言える。** オンボーディングがシンプル
- **CLI は bitbank API を直接叩く設計なので、共有コードは最小限。** リポ分離のコストが低い

## 影響

- API 型定義の一部が二重管理になる（許容範囲）
- 共通化の必要性が出たら `bitbank-core` パッケージ切り出しを検討する
