# 戦略パターン & 計算方法リファレンス

## 戦略テンプレート

### 1. SMA クロス戦略

```
エントリー（買い）: SMA(短期) が SMA(長期) を上抜け
イグジット（売り）: SMA(短期) が SMA(長期) を下抜け
デフォルトパラメータ: 短期=20, 長期=50
```

### 2. RSI 逆張り戦略

```
エントリー（買い）: RSI(14) が 30 を下回った後、30 を上抜け
イグジット（売り）: RSI(14) が 70 を超えた、または一定利益率に到達
デフォルトパラメータ: 期間=14, 買い閾値=30, 売り閾値=70
```

### 3. MACD クロス戦略

```
エントリー（買い）: MACD ラインがシグナルラインを上抜け（ゴールデンクロス）
イグジット（売り）: MACD ラインがシグナルラインを下抜け（デッドクロス）
デフォルトパラメータ: 短期EMA=12, 長期EMA=26, シグナル=9
```

### 4. ボリンジャーバンド ブレイクアウト戦略

```
エントリー（買い）: 価格がロワーバンドを下回った後、ミドルバンドに回帰
イグジット（売り）: 価格がアッパーバンドに到達、またはミドルバンドを下抜け
デフォルトパラメータ: 期間=20, σ=2
```

### 5. 複合戦略（トレンド+モメンタム）

```
フィルター: SMA(200) の上にいるときのみトレード（上昇トレンド）
エントリー（買い）: RSI(14) < 40 かつ MACD ヒストグラム > 0
イグジット（売り）: RSI(14) > 70 または SMA(200) を下抜け
```

## P&L（損益）計算方法

### 1トレードの損益

```
買いの場合:
  損益(JPY) = ポジションサイズ × (売値 - 買値) / 買値 - 手数料

手数料:
  エントリー手数料 = ポジションサイズ × 手数料率
  イグジット手数料 = (ポジションサイズ + 含み損益) × 手数料率
  合計手数料 = エントリー手数料 + イグジット手数料
```

### 資金推移

```
初期資金 = 1,000,000 JPY（デフォルト）
各トレード後の資金 = 前回資金 + トレード損益 - 手数料
```

### 複利 vs 単利

- **デフォルト: 複利。** 各トレードで現在の全資金を投入
- ユーザーが固定額を指定した場合は単利

## データリーク回避

バックテストは「未来情報を使わない」ことが大前提。`shift(1)` を一つ忘れるだけで、
存在しないエッジが結果に現れる。同じデータ・同じロジックでも、シグナル生成に使った
足の close で約定してしまえば「未来情報を使ったトレード」になる。

### 基本ルール

- **シグナルは `t` のデータで計算する。** 足 `t` が確定した時点で得られる情報のみ使用
- **約定は `t+1` の始値で行う。** シグナル足の close で約定してはいけない（同足リーク）
- **指標の rolling 計算は過去のみ参照する。** 移動平均・RSI・MACD などの計算窓は `t` 以前のデータに閉じる

### やりがちなリーク例

1. **rolling 計算が future-leaking**
   - pandas の `rolling(..., center=True)` は窓を中心に置くため、未来のデータを使う
   - バックテストでは `center=False`（デフォルト）を必ず使う
   - 同様に `expanding()` や `ewm()` も将来データが混入しない実装か確認する

2. **将来リターンを `shift(-1)` する向きを間違える**
   - 「次の足のリターン」を作る目的で `shift(-1)` するのは正しいが、それを **シグナルの説明変数として使う** と未来情報を使ったことになる
   - 説明変数（指標）は `shift(1)` 以上、目的変数（リターン）が `shift(-1)` という対応を必ず確認する

3. **データ結合時にタイムゾーンずれで未来が混入**
   - JST と UTC を混在させたまま `merge_asof` などで結合すると、9 時間ぶん未来のデータが現在の行に紐付く
   - bitbank API は UTC 基準。結合前に必ずタイムゾーンを揃える

4. **指標を全期間で正規化してからスライス**
   - z-score や min-max 正規化を全期間に対して計算してからバックテスト窓を切ると、過去時点で「未来の最小値・最大値」を知っていることになる
   - 正規化も rolling で過去のみ参照する形にする

### 検証手段

SKILL.md の Step 3.5「リークサニティチェック」を参照。`shift(1)` あり版となし版を
並べて、結果差が極端に大きくないかを確認する。差が 2 倍を超える場合はロジックを
止めて再確認する（閾値はあくまで目安）。

## リスク指標の算出

### 勝率

```
勝率 = 勝ちトレード数 / 全トレード数 × 100
```

### プロフィットファクター

```
プロフィットファクター = 総利益 / 総損失（絶対値）
```

- 1.0 以上: 利益が損失を上回る
- 2.0 以上: 良好

### 最大ドローダウン

```
各時点の資金残高を記録
ピーク = それまでの最大資金残高
ドローダウン = (ピーク - 現在資金) / ピーク × 100
最大ドローダウン = 全期間のドローダウンの最大値
```

### シャープレシオ（簡易版）

```
各トレードのリターン率を計算
シャープレシオ = 平均リターン率 / リターン率の標準偏差 × √(年間トレード数)

年間トレード数の目安:
  日足: 365
  4時間足: 365 × 6
  1時間足: 365 × 24
```
