# 銘柄間相関分析 ガイド

`SKILL.md` の各分析項目について、**理論的背景・計算式の留意点・実務での読み方**
を補足する。モデルが計算ミスをしないことと、結果を解釈にどう翻訳するかを
両方カバーする。

## 1. なぜリターンで相関を取るのか — 価格相関とリターン相関

### 1.1 数学的背景

価格そのもの `P_X[t]`, `P_Y[t]` の系列は **非定常**（unit root を持つ
ランダムウォーク的時系列）。非定常な 2 系列の相関は、
共通のトレンド成分があるだけで容易に **|r| > 0.9** になる
（spurious correlation, 偽相関）。

代表的な例:

- 2 つの独立なランダムウォークでも、サンプル数を増やすほど標本相関が
  ゼロから乖離していく（Yule, 1926）
- 暗号資産の価格は強気期に揃って上昇するため、BTC と XRP の **価格相関**
  は 0.95 を超えることもあるが、これは「同じ強気トレンドに乗っている」
  ことの反映でしかない

リターン `r[t] = log(P[t] / P[t-1])` は **差分系列** で、ほぼ定常に近づく。
平均回帰せず、自己相関が小さく、スケールが揃う。
**リターン相関は「同じ瞬間にどれだけ同じ方向に動くか」の真の指標** になる。

### 1.2 実務での意味

- 価格相関 0.97 は「同じトレンドに乗っている」だけ。リスク分散の判断材料に
  ならない
- リターン相関 0.81 は「日々の値動きの 65%（= 0.81²）が共通因子で説明できる」
  と解釈できる
- 本 skill は Step 1 で価格相関を **対比用に** 出すが、Step 2 以降は
  すべてリターン相関で議論する

## 2. Pearson と Spearman の使い分け

### 2.1 定義

- **Pearson r**: 標準化した値の積の平均（線形相関）

  ```
  r = mean( ((x - mean_x) / std_x) * ((y - mean_y) / std_y) )
  ```

- **Spearman r**: 順位（rank）に変換してから Pearson を計算（順位相関）

### 2.2 使い分けと差分の解釈

| 状況 | 含意 |
|------|------|
| `\|P − S\| < 0.05` | 関係がほぼ線形。Pearson で十分 |
| `P > S` に大きく振れる（差 > +0.05） | 少数の大変動足が Pearson を押し上げ。外れ値の影響が強い |
| `P < S` に大きく振れる（差 < -0.05） | 順位レベルでは連動するが線形回帰には乗らない。非線形変換（log, winsorize）を検討 |

暗号資産では **暴落足や急騰足が外れ値として作用** することが多く、
Pearson > Spearman の差が出やすい。Step 6 のトリミングでこの効果を確認する。

### 2.3 p 値を使わない理由

- 暗号資産時系列は **i.i.d. でない**（ファットテール + ボラクラスタリング）
- 古典的 Pearson の p 値は i.i.d. 前提のため **過小評価（楽観バイアス）**
- 「N=730 なので p < 0.001 です」という主張は鵜呑みにしない
- 本 skill は **r の絶対値とローリング相関の安定性** を主軸にし、p 値は出さない

## 3. β の導出と解釈

### 3.1 導出

最小二乗回帰 `Y = α + β X + ε` の β は:

```
β_{Y on X} = Cov(X, Y) / Var(X)
            = (r_{X,Y} × σ_X × σ_Y) / σ_X²
            = r_{X,Y} × σ_Y / σ_X
```

つまり β は **「相関の強さ × ボラ比」**。同じ相関でも Y のボラが大きければ β
は大きくなる。

### 3.2 非対称性

```
β_{Y on X} = r × σ_Y / σ_X
β_{X on Y} = r × σ_X / σ_Y
```

両者は σ 比の逆数の関係にあり、`σ_X ≠ σ_Y` なら **必ず別値**。

例: BTC σ = 2.5%, ETH σ = 3.0%, r = 0.81 のとき

- β_{ETH on BTC} = 0.81 × 3.0 / 2.5 = **0.972**（BTC が 1% 動くと ETH が 0.97%）
- β_{BTC on ETH} = 0.81 × 2.5 / 3.0 = **0.675**（ETH が 1% 動くと BTC が 0.68%）

行列を「対称化」すると意味が壊れる。本 skill は両方向の β を出す。

### 3.3 実務での解釈

| β | 含意 |
|---|------|
| > 1 | 増幅。X より Y の方が振れ幅が大きい |
| ≈ 1 | 同程度の振れ幅 |
| < 1 | 減衰。Y の方が静か |
| < 0 | 逆方向（暗号資産同士では稀） |

「ETH ヘッジに BTC を使いたい」なら β_{ETH on BTC} が必要。BTC ロング 1 単位に
対して ETH 0.97 単位ショートが理論ヘッジ比。

## 4. ローリング相関のトレードオフ

### 4.1 窓サイズの選択

| 窓 | 特徴 |
|----|------|
| 短い（例: 1day で 30 本 ≒ 1 ヶ月）| ノイズ多いが直近のレジーム変化に追従 |
| 長い（例: 1day で 90 本 ≒ 3 ヶ月）| 安定するが反応が遅行 |

本 skill のデフォルトは **30 / 90 の 2 窓** で、ノイズと滑らかさを両方見る。

### 4.2 trailing window 厳守

- `center=True`（中心合わせ）は未来データを参照する → リーク
- 全期間正規化も同様に NG（過去判断に未来統計が混入）
- 本 skill のすべての rolling は **末尾合わせ（trailing window）**

### 4.3 全期間 r との乖離

- 「全期間 0.8 なのに、ある月だけ 0.4 まで落ちる」のは普通に起こる
- 最大乖離が `> 0.3` あれば「相関は時期によって大きくブレる」と読む
- 平均と直近を併記して **「いま」の連動度** が全期間平均から外れていないか確認

## 5. 環境別相関の条件付けバイアス

### 5.1 なぜバイアスが出るか

条件銘柄 X の `r_X > 0` の足だけを使って `corr(r_X, r_Y)` を計算すると、
**X の符号が片側に固定されている** ので、自己相関的に r が高く出る。
これは Y の挙動と無関係に発生する **機械的バイアス**。

そのため、本 skill は **「条件銘柄を含むペア」** にバイアス警告を必ず付ける。
解釈の主軸は **条件銘柄を含まないペア**（例: 条件 = BTC のとき ETH-XRP）に置く。

### 5.2 なぜ環境別相関を見る価値があるか

- 暗号資産は **下落時に相関が上がる**（クラッシュ時に分散効果が消える）
  という stylised fact がある
- 平時は ETH-XRP が 0.5 でも、BTC が大幅下落する局面では 0.7〜0.8 に
  跳ね上がることがある
- ポートフォリオ管理では **「最も分散効果が必要なときに分散効果が消える」**
  ので、平均相関だけ見ると分散効果を過大評価する

### 5.3 出力ルール

- `down − up >= +0.10`: 下落時に相関上昇。「分散効果が崩れる」傾向
- `down − up <= -0.10`: 下落時に相関低下（暗号資産では稀）
- `|down − up| < 0.10`: 環境による差は小さい

ペア数 = 2 では条件銘柄を含むペアしか存在せず、解釈不能。**Step 7 をスキップ**
してその旨を出力する。

## 6. ラグ相関と多重比較

### 6.1 計算

```
corr_lag[k] = Pearson(r_X[t], r_Y[t + k])
```

- `lag = 0`: 同時相関（Step 2 の Pearson と一致）
- `lag > 0`: X が Y より先行（X の動きが k 足後の Y と相関）
- `lag < 0`: Y が X より先行

### 6.2 なぜ「ピークが lag = 0 ならリードラグなし」と保守的に判定するか

- 暗号資産の主要ペアは **基本的に同時動作**。BTC, ETH, XRP は同じグローバル
  な情報（マクロイベント、規制、流動性）に同時反応する
- リードラグが出るとしても **数秒〜数分**。1 day 足ではほぼ検出不能
- ノイズと多重比較で偶然のピークが lag ≠ 0 に立ちやすい
- 「**lag = 0 のピークが他より顕著に低くなければ、リードラグなしと結論する**」
  が本 skill の保守的方針

### 6.3 多重比較

- 1 ペアで `(2k + 1)` 個の lag を同時に検定
- ペア数 × lag 数で偶然のピークが出る確率は無視できない
- Bonferroni: `α / m` （m = 全検定数）。厳しすぎる場合は FDR (Benjamini-Hochberg)
- **本 skill は補正後 p 値を出さない**。代わりに:
  - **r の絶対値を主軸**（小さな |r| はピークでも採用しない）
  - **安定性チェック**（4 サブ期間で符号が一致しない → 警告）
  で偶然のピークを排除する

### 6.4 安定性チェックの考え方

全期間で `lag = +1` に r = 0.15 のピークが出ても、4 サブ期間に分けたとき:

- 4 サブ期間すべてで r > 0 → 持続的な傾向
- 4 サブ期間で 2 つ以上が r < 0 → 不安定。偶然の可能性

判定は `lag = +1` の符号で簡易化する。サブ期間数はデフォルト 4。
データ少ない時は 3 に減らしてもよい（ユーザー指定可）。

## 7. 多重比較補正の本 skill での扱い方針

| 補正 | 概念 | 本 skill での扱い |
|------|------|------------------|
| Bonferroni | `α / m` で個別有意水準を下げる | 採用しない（p 値を出さないため） |
| Holm | Bonferroni をステップワイズで緩和 | 同上 |
| FDR (BH) | 偽発見率を制御 | 同上 |

**理由**: 本 skill のゴールは「記述」であり、「有意/非有意」を二値で判定しない。
代わりに以下で偽発見を抑える:

1. **r の絶対値の閾値**（弱相関は採用しない）
2. **ローリング相関の安定性**（時期によって符号反転するなら採用しない）
3. **環境別相関の整合性**（up と down で正負が逆転するなら警告）
4. **ラグ相関の安定性チェック**（4 サブ期間で符号一致を要求）

p 値ベースの厳密な統計検定が必要なら、別途 R / Python で実施するのが筋。

## 8. 平均相関と分散投資効果の試算

### 8.1 ポートフォリオ分散の式

n 銘柄を等ウェイト `w = 1/n` で持つポートフォリオの分散:

```
Var(P) = (1/n²) × Σ Var(r_i) + (1/n²) × Σ_{i≠j} Cov(r_i, r_j)
       = (1/n) × σ̄² + ((n-1)/n) × σ̄² × ρ̄
```

ここで `σ̄²` は平均分散、`ρ̄` は平均相関。よって:

```
σ_P / σ̄ = √( (1/n) + ((n-1)/n) × ρ̄ )
```

### 8.2 平均相関と分散減衰率

n = 3 銘柄の場合:

| 平均相関 ρ̄ | σ_P / σ̄ | 分散減衰 |
|-----------|----------|---------|
| 0.0 | 0.577 | -42% |
| 0.3 | 0.730 | -27% |
| 0.5 | 0.816 | -18% |
| 0.7 | 0.894 | -11% |
| 0.9 | 0.966 | -3% |
| 1.0 | 1.000 | 0% |

n → ∞ のとき `σ_P / σ̄ → √ρ̄`。つまり **平均相関 0.7 では n をいくら増やしても
ボラを √0.7 ≈ 84% より下げられない**。

### 8.3 本 skill のサマリー閾値

- 平均相関 > 0.7 → サマリーで「**分散効果は限定的**」とコメント
- 平均相関 ≤ 0.5 → 「分散効果あり」（明示はしないが警告は出さない）
- 平均相関 ≤ 0.3 → 「分散効果が大きい」（暗号資産同士では稀）

暗号資産同士の分散には限界があるため、本格的に分散したいなら **異なる
アセットクラス**（株式、債券、コモディティ）を組み合わせる必要がある
（本 skill のスコープ外）。

## 10. inner join とサンプル数の問題

### 10.1 整列方針

複数ペアを比較するには **同じタイムスタンプ** での close が必要。本 skill は
**inner join**（全ペアに存在する足だけ残す）を採用する。

理由:

- left join + 欠損補完（forward fill 等）はリターン計算で 0 リターンを生み、
  相関を歪める
- pairwise（ペアごとに別の inner join）は行列の整合性が崩れる

### 10.2 サンプル激減のリスク

- 上場日が違うペアを混ぜると、共通期間が短くなる
  （例: 古参の BTC と新規上場の SUI を混ぜると SUI の上場日以降に制限）
- 取引停止期間がある minor ペアを混ぜると、欠損足で共通期間が穴あき
- **事前に data-verification skill** で各ペアの本数・欠損を確認推奨

inner join 後のサンプル数 N を出力に必ず含め、N < 100 程度なら警告。

## 12. 本 skill が扱わないこと

明確化のため、以下は **本 skill のスコープ外**:

- **売買判断・閾値最適化**: 売買ロジックは出さない。「相関 < 0.5 で買う」等
- **戦略バックテスト**: コスト込みの PnL は backtest skill
- **将来予測**: ラグ相関は記述であり「BTC が 1 日先行するから XRP を先回り買い」
  といった結論は出さない
- **異種アセット**: bitbank 現物 JPY ペアのみ。USDT クロス・株式・債券・
  コモディティとの相関は扱わない（外部データ取得の仕組みを持たない）
- **コピュラ・尾部依存**: ファットテール時の依存構造は別フレームワークが必要
- **回帰のフルセット**: α、p 値、信頼区間、residual diagnostics は出さない。
  β は σ 比 × r で簡易計算

これらが必要なら、本 skill の出力を起点に R / Python で別途実施する。
