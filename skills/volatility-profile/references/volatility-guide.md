# ボラティリティ・プロファイル ガイド

SKILL.md の各分析項目について、計算式・解釈・実務での使い方を補足する。
モデルが計算ミスをしないことと、結果を実務判断にどう翻訳するかを両方カバーする。

## 1. リターン基本統計量

### 1.1 リターンの定義

本 skill は **対数リターン**で統一する:

```
r[t] = log(close[t] / close[t-1])
```

- 加法性が成立する（n 期間の合計 = 各期間の和）
- 対称性: 「+10% → -10%」が対数リターンでは絶対値で同じ
- 単純リターン `(close[t] - close[t-1]) / close[t-1]` を望まれた場合のみ切替

### 1.2 各統計量の計算と解釈

| 統計量 | 計算 | 解釈 |
|-------|-----|------|
| mean | 算術平均 | 期間内のドリフト。短期足ではほぼゼロに近い |
| std (σ) | 標本標準偏差 | ボラティリティ。BB の σ や VaR の入力 |
| skewness | 標準化 3 次モーメント | 0=対称、+=右裾長い、-=左裾長い。暗号資産は通常マイナス（暴落の方が深い） |
| **excess kurtosis** | 標準化 4 次モーメント − 3 | 0=正規分布、+=ファットテール。暗号資産は通常 5〜20 |

**超過尖度の式（モデル直接実装時）:**

```
mu  = mean(r)
sd  = std(r)            # 標本 std
ek  = mean(((r - mu) / sd) ** 4) - 3
```

**注意:** pandas の `Series.kurtosis()` は **超過尖度（excess kurtosis）** を返す
（既に -3 されている）。「kurtosis = 3 が正規分布」と書く資料は通常尖度
（Pearson 定義）で、本 skill の出力ラベルとは異なる。混在させない。

### 1.3 暗号資産での典型値

- BTC/JPY 1hour（過去 1 年）: 歪度 -0.1〜-0.3、超過尖度 5〜10
- ETH/JPY 1hour: 歪度 -0.2〜-0.5、超過尖度 6〜15
- minor アルト 1hour: 超過尖度 20 超もありうる

超過尖度がマイナスや 0 付近に出たら **計算式の二重補正**（-3 を二重に引いた等）を疑う。

## 2. ファットテール倍率テーブル

### 2.1 理論頻度（定数）

正規分布で `|z| > k` となる両側確率（百分率）:

| k | 両側確率 |
|---|---------|
| 2 | 4.5500% |
| 3 | 0.2700% |
| 4 | 0.0063% |
| 5 | 0.000057% |

これらは標準正規分布の累積確率から定数で導出できる。
本 skill では `scipy.stats.norm` を呼ばず、**上記定数を直接使う**。

### 2.2 実測頻度の計算

```
z[t] = (r[t] - mean(r)) / std(r)
empirical[k] = (|z| > k).sum() / N * 100   # 百分率
```

### 2.3 倍率の解釈

| 倍率 | 含意 |
|-----|------|
| 1.0 付近 | 正規分布で十分近似できる（暗号資産では稀） |
| 2〜5x | 緩いファットテール。VaR の正規前提は若干過小評価 |
| 5〜20x | 強いファットテール。`±3σ` で「滅多にない」と扱うのは危険 |
| 20x 超 | 極端なファットテール。ヒストリカル VaR / EVT 系の手法を検討 |

### 2.4 VaR への含意

正規分布前提の 99% VaR は約 2.33σ。実測の ±2.33σ 頻度が理論の数倍なら、
「99% VaR を超える損失は年に 2〜3 回」のはずが、実際は **年に 10 回以上** 起きる
可能性がある。サイジングの安全係数として、超過尖度 × √2 程度のバッファを
ストップ幅に乗せるのが保守的（経験則）。

## 3. 時間帯別出来高プロファイル

### 3.1 集約方法

```
hour_utc = floor((timestamp_ms / 1000 / 3600) % 24)
profile[h] = mean(volume where hour_utc == h)   for h in 0..23
```

- タイムスタンプは bitbank 仕様で **UTC ミリ秒**
- bitbank の取引参加者は JST 基準で動くため、**UTC と JST を併記**:
  - JST hour = (UTC hour + 9) % 24

### 3.2 bitbank の時間帯特性（JPY ペア）

| JST 時間帯 | UTC 時間帯 | 流動性 | 背景 |
|-----------|-----------|-------|------|
| 09:00〜11:00 | 00:00〜02:00 | 中 | 日本市場の朝。出社後の取引 |
| 13:00〜15:00 | 04:00〜06:00 | **低** | 昼休み明け。アジア時間で薄い |
| 16:00〜19:00 | 07:00〜10:00 | 中 | 欧州市場序盤 |
| 21:00〜23:00 | 12:00〜14:00 | **高** | NY 市場序盤 + 日本の夜の個人 |
| 22:00〜00:00 | 13:00〜15:00 | **最高** | NY 市場活発 + 経済指標発表多い |
| 02:00〜06:00 | 17:00〜21:00 | 中〜低 | NY 後半〜引け |

USDT クロス（海外取引所中心）と違い、bitbank の JPY ペアは **JST 22〜0 時に偏る**
傾向が強い。日中時間帯（JST 13〜15）の大口注文は、薄い板を叩いて滑りやすい。

### 3.3 実務での使い方

- **大口エントリー**: 高流動性時間帯を狙う。スリッページが小さい
- **ストップ注文の設置**: 低流動性時間帯にストップを置くと、薄い板を叩いて
  本来の水準より深く約定する可能性がある。指値ストップとの併用を検討
- **指標発表時**: NY 時間（JST 21:00〜23:30）に米雇用統計・FOMC 等。
  事前にポジション調整するなら、流動性のある時間帯で

## 4. 出来高 × |リターン| の相関

### 4.1 計算

```
abs_return = |r[t]|
volume     = ohlcv[t][4]   # 数値変換済み

pearson  = corr(abs_return, volume)              # ピアソン
spearman = corr(rank(abs_return), rank(volume))  # スピアマン
```

両系列に NaN（リターン先頭等）が含まれないよう揃えてから計算する。

### 4.2 解釈

- **Pearson r > 0.5**: 出来高と価格変動が強く連動。出来高サージ → 大変動の予兆
- **Pearson r 0.2〜0.5**: 中程度の連動。日常レベルでは弱いが、イベント時は連動
- **Pearson r < 0.2**: ほぼ独立。出来高をシグナルに使いにくい

### 4.3 Pearson と Spearman の差

- **|Pearson - Spearman| < 0.05**: 関係はほぼ線形。シンプルな線形モデルでよい
- **Pearson > Spearman に大きく振れる（差 > 0.1）**:
  少数の外れ値（大口約定 + 大変動の足）が Pearson を押し上げている。
  ロバスト推定を使うか、外れ値の足を別扱いにする
- **Spearman > Pearson に大きく振れる（差 > 0.1）**:
  順位レベルでは連動しているが、線形回帰には乗りにくい。
  非線形変換（log volume 等）を検討

### 4.4 実務での含意

- 「出来高急増 → 大変動」というアラートを組むなら、Pearson > 0.5 が前提
- Spearman > Pearson のとき、`log(volume)` を特徴量にすると線形モデルが機能しやすい

## 5. √T スケーリング比

### 5.1 理論的背景

リターンが **i.i.d.（独立同分布）** なら、n 期間合計リターンの標準偏差は
`σ × √n` になる（中心極限定理の帰結）。これが **√T スケーリング則**。

```
σ(n 期間合計) = σ(1 期間) × √n      （i.i.d. のとき）
```

時間倍率 n は、長期足の 1 本が短期足の何本ぶんかで決まる:

| 短期 → 長期 | n | √n |
|------------|---|----|
| 1min → 1hour | 60 | 7.746 |
| 5min → 1hour | 12 | 3.464 |
| 15min → 1hour | 4 | 2.000 |
| 30min → 1hour | 2 | 1.414 |
| 1hour → 4hour | 4 | 2.000 |
| 1hour → 1day | 24 | 4.899 |
| 4hour → 1day | 6 | 2.449 |
| 8hour → 1day | 3 | 1.732 |
| 12hour → 1day | 2 | 1.414 |
| 1hour → 1week | 168 | 12.961 |
| 1day → 1week | 7 | 2.646 |

### 5.2 比率の解釈

```
比率 = σ_long / (σ_short × √n)
```

| 比率 | 解釈 |
|-----|------|
| 0.9 〜 1.1 | √T 則がほぼ成立。リターンは独立に近い |
| > 1.1 | 長期の方がボラ大。**ボラクラスタリング**（ARCH 効果）。連続する大変動が同方向に持続 |
| < 0.9 | 平均回帰的。短期の振れが長期では相殺。レンジ相場 |

### 5.3 暗号資産での適用上の注意

- **期間ミスマッチに弱い:** 短期と長期で取得期間がズレるとレジーム差で比率が
  簡単に 0.5〜2.0 を超える。両足の `--from / --to` を揃える
- **データ本数の最低水準:** 短期 1,000 本以上、長期 100 本以上が目安。
  少なすぎると σ の推定誤差で比率が暴れる
- **1month は不定間隔。** √T の母数として使わない

### 5.4 実務での含意

- **比率 ≈ 1.0:** 短期 σ から長期 VaR を `× √n` で外挿しても安全
- **比率 > 1.1:** 短期 σ × √n は長期リスクを **過小評価**。
  ストップ幅・サイジングを保守的に。GARCH 系モデルの導入を検討
- **比率 < 0.9:** 短期で慌てる必要は少ない。レンジ取り戦略の余地

## 6. 出力結果の実用への翻訳

各項目をストップ幅・サイジングにどう使うかの実践例。

### 6.1 ストップ幅の決定

- **ベース**: 直近 σ（短期足）の **2〜3 倍** をストップ幅の出発点に
- **超過尖度補正**: 超過尖度が 10 を超えるなら、上記に **× 1.2〜1.5** のバッファ
- **時間帯補正**: 低流動性時間帯にエントリーするなら、さらに **× 1.2** 程度の余裕を
- **例**: BTC/JPY 1hour σ = 0.56% → ベース ±1.7%（3σ）。
  超過尖度 7.6 でバッファ 1.3x → **±2.2%**

### 6.2 ポジションサイジング

- **正規前提の Kelly**: ファットテールで過大ポジションを推奨してくる。
  超過尖度 > 5 なら **Half-Kelly 以下** を推奨
- **VaR ベース**: 99% VaR を計算するなら、**ヒストリカル VaR**（実測分位点）を
  使う。正規分布前提だとファットテール倍率ぶん過小評価される

### 6.3 時間帯戦略

- **大口エントリー / 利確**: 高流動性時間帯（JST 22:00〜0:00）を選ぶ
- **ストップロス設置**: 低流動性時間帯（JST 13:00〜15:00）に **指値ストップ** を
  置くと、滑った価格で約定するリスクを抑えられる
- **指標発表前後**: NY 時間の指標発表（JST 21:30〜23:00 が多い）前にポジション
  サイズを下げる、もしくは事前にヘッジ

### 6.4 √T 比率の活用

- **比率 ≈ 1.0**: 「日次 σ ≈ 時間 σ × √24」で長期リスクを推定可能
- **比率 > 1.1**: 短期 σ から外挿せず、**長期足で直接 σ を測る** べき。
  GARCH 等のクラスタリングモデルの導入検討
- **比率 < 0.9**: レンジ相場。逆張り戦略の親和性が高い時期
